# اطلاعات هزینه دیتابیس PostgreSQL در Scalingo

## وضعیت فعلی:

✅ **پلن فعلی**: `postgresql-sandbox` - **رایگان در دوره آزمایشی**

## جزئیات هزینه:

### 1. **دوره آزمایشی (Trial Period)**
- Scalingo یک **دوره آزمایشی 30 روزه رایگان** ارائه می‌دهد
- در این دوره، همه سرویس‌ها (از جمله دیتابیس) رایگان هستند
- **نیازی به کارت اعتباری نیست**

### 2. **بعد از دوره آزمایشی**

#### گزینه 1: حذف دیتابیس (اگر نمی‌خواهی هزینه پرداخت کنی)
```bash
scalingo --app mywebsite addons-remove mywebsite_2040
```
⚠️ **نکته**: با حذف دیتابیس، دوباره از SQLite استفاده می‌شود و داده‌ها در هر deployment پاک می‌شوند.

#### گزینه 2: استفاده از دیتابیس خارجی (رایگان)
می‌توانی از سرویس‌های رایگان دیتابیس استفاده کنی:
- **Supabase** (رایگان تا 500MB)
- **Railway** (رایگان تا 5GB)
- **Neon** (رایگان تا 3GB)
- **ElephantSQL** (رایگان تا 20MB)

#### گزینه 3: ادامه استفاده از Scalingo PostgreSQL
- **postgresql-sandbox**: ممکن است بعد از trial هزینه داشته باشد
- **postgresql-starter-512**: پلن پولی (512MB)
- هزینه‌ها بر اساس استفاده محاسبه می‌شوند

## توصیه:

1. **الان**: از دیتابیس رایگان استفاده کن (در دوره آزمایشی)
2. **بعد از 30 روز**: 
   - اگر می‌خواهی رایگان بمانی → از دیتابیس خارجی استفاده کن (Supabase پیشنهاد می‌شود)
   - اگر می‌خواهی در Scalingo بمانی → پلن پولی انتخاب کن

## راه‌حل رایگان دائمی:

### استفاده از Supabase (رایگان):

1. **ثبت‌نام در Supabase**: https://supabase.com
2. **ایجاد پروژه جدید**
3. **کپی کردن Connection String**
4. **تنظیم در Scalingo**:
   ```bash
   scalingo --app mywebsite env-set DATABASE_URL="postgresql://..."
   ```
5. **حذف دیتابیس Scalingo** (اگر می‌خواهی):
   ```bash
   scalingo --app mywebsite addons-remove mywebsite_2040
   ```

## بررسی هزینه فعلی:

برای بررسی هزینه‌های فعلی:
- به **Scalingo Dashboard** برو
- بخش **Billing** را چک کن
- یا از CLI:
  ```bash
  scalingo --app mywebsite billing
  ```

## خلاصه:

- ✅ **الان**: رایگان (دوره آزمایشی 30 روزه)
- ⚠️ **بعد از 30 روز**: باید تصمیم بگیری:
  - استفاده از دیتابیس خارجی رایگان (Supabase)
  - یا پرداخت هزینه Scalingo PostgreSQL
  - یا برگشت به SQLite (با از دست دادن داده‌ها)
